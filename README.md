# Динамические структуры данных в языке Go
---

## Описание реализации

### Предметная область: геометрические объекты (пакет `figures`)

| Компонент           | Назначение                                                         |
|---------------------|--------------------------------------------------------------------|
| `Shape` (интерфейс) | Контракт с методами `Draw()`, `GetName() string`, `Area() float64` |
| `BaseShape`         | Базовая структура, инкапсулирующая поля `color` и `rotation`       |
| `Circle`            | Круг. Площадь: `π·r²`                                              |
| `Rectangle`         | Прямоугольник. Площадь: `w·h`                                      |
| `Square`            | Квадрат. Площадь: `a²`                                             |
| `Triangle`          | Треугольник. Площадь: `a·h / 2`                                    |
| `Rhombus`           | Ромб. Площадь: `d₁·d₂ / 2`                                         |

**Особенности:**

- Поворот нормализуется в диапазон `[0°; 360°)` функцией `math.Mod` с корректной обработкой отрицательных значений.
- Встраивание `BaseShape` обеспечивает наследование методов `SetColor`, `GetColor`, `SetRotation`, `GetRotation` всеми фигурами.

### Двусвязный список (`LinkedList`)

В основе реализации двусвязного списка лежит механизм явного связывания объектов через указатели. В отличие от массивов, где элементы расположены в памяти смежно, узлы списка могут находиться в произвольных участках кучи (heap), а их логическая последовательность обеспечивается исключительно ссылками.

##### Структура узла и роль указателей

Каждый узел списка (`Node`) содержит три ключевых компонента:

```go
type Node struct {
    value Shape         // ссылка на фигуру (интерфейсы в Golang всегда передаются по ссылке)
    prev  *Node         // указатель на предыдущий узел
    next  *Node         // указатель на следующий узел
}
```

Поля `prev` и `next` являются указателями на структуру того же типа (`*Node`). Это создаёт рекурсивную связь: каждый узел «знает» о своих соседях, образуя цепочку. Именно наличие двух указателей делает список двусвязным, позволяя осуществлять обход в обоих направлениях и выполнять удаление узла за O(1) при наличии прямой ссылки на него.

Ключевой особенностью данной реализации является поле `value` типа `Shape`. Поскольку `Shape` — это интерфейс, а не конкретный тип, в данное поле может быть помещён указатель на любую структуру, реализующую этот интерфейс (`Circle`, `Rectangle`, `Triangle` и т.д.).

Реализованы методы:

- `AddFirst(Shape)` - добавление в начало списка;
- `AddLast(Shape)` - добавление в конец списка;
- `Remove(*Node)` - удаление узла с корректным обновлением `head`/`tail`;
- `Print()` - вывод всех фигур;
- `PrintAreas()` - вывод только площадей.

### Бинарное дерево поиска (`BinaryTree`)

Бинарное дерево поиска использует тот же фундаментальный механизм связывания через указатели, но организует их в иерархическую (древовидную) структуру вместо линейной цепочки.

##### Структура узла дерева
```go
type TreeNode struct {
    value Shape         //  ссылка на фигуру
    left  *TreeNode     // указатель на левый узел
    right *TreeNode     // указатель на правый узел
}
```
Аналогично списку, поля `left` и `right` являются указателями на структуру того же типа (`*TreeNode`), что обеспечивает рекурсивную природу дерева: каждое поддерево само является деревом. Однако семантика связей здесь иная:

`left` указывает на узел, значение которого строго меньше текущего;

`right` указывает на узел, значение которого больше или равно текущему.

Поле `value` имеет тип интерфейса `Shape`, что позволяет хранить в одном дереве разнородные геометрические объекты. Упорядочивание производится не по типу фигуры, а по числовому ключу — площади `Area()`, вычисляемой полиморфно.

Дерево упорядочено по возрастанию площади фигуры. Реализованы методы:

- `Insert(Shape)` - рекурсивная вставка;
- `Search(float64)` - поиск фигуры по площади с использованием эпсилон-сравнения (`|a − b| < 10⁻⁹`) для корректной работы с типом `float64`;
- `FindMin()` / `FindMax()` - поиск экстремумов;
- `Height()` - вычисление высоты дерева;
- `Size()` - подсчёт количества узлов;
- `FindInRange(min, max)` - поиск фигур в заданном диапазоне площадей;
- `ToSlice()` / `PrintInOrder()` - in-order обход.

## Инструкция по запуску

**Требования:** Go версии 1.18 и выше.

```bash
# инициализация модуля (при отсутствии go.mod)
go mod init dynimic-data-structure

# запуск программы
go run main.go
```
